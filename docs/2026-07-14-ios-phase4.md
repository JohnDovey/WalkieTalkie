## Build prerequisites

1. Source the volume env (Xcode / JDK / Android already set):

   ```bash
   source ~/source-john-dovey.sh --quiet
   source tools/go-env.sh
   ```

2. Bind the shared Go core (gitignored output â€” same pattern as Android's AAR):

   ```bash
   ./tools/gomobile-bind-ios.sh
   # â†’ ios/Frameworks/Core.xcframework  (ios + iossimulator)
   ```

   Requires `golang.org/x/mobile` in `core/` (tool directive / `go get -tool`). Uses `-ldflags="-checklinkname=0"` and binds `./mobile ./media` together (same reasons as Android).

3. Build vendored libopus (AVFoundation has no Opus encoder/decoder):

   ```bash
   ./tools/build-opus-ios.sh

   # Cache: /Volumes/JohnDovey/tmp/walkietalkie-opus-ios
   ```

   Configured with `--disable-asm` so Apple's assembler does not choke on GNU ARM assembly.

4. Regenerate the Xcode project after `project.yml` changes:

   ```bash

   ```

5. Signing: copy `ios/Config/Local.xcconfig.example` â†’ `ios/Config/Local.xcconfig` (gitignored) and set `DEVELOPMENT_TEAM`. Automatic signing. ## Opus choice

No AVFoundation Opus. Static **libopus 1.5.2** packaged as `Opus.xcframework`, with a tiny C bridge (`OpusCodec.c`) used from Swift. Target: 48 kHz mono, 20 ms frames, 32 kbps â€” aligned with Android `OpusMicSource`.
